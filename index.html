How this can help You?

If you have ever been working with audio files on some web applications - we bet you had encoutered bunch of problems without easy-to-apply solutions...

Our experience is quite similar and because quite a few times we were creating portals or websites that were focused around audio - we eventually decided to create processing tools quicken things up in the future

As it seems obvious - "we're not the only ones" having these problems, so we kindly invite you to help yourself with Audiomatic.

01

Bloody waveforms generation...

If you've ever tried to fetch a waveform from an audio file - you definitely realised this it isn't a trivial task and there are no useful tools for that...


Not anymore - things just changed:)

02

Elastic time?! Definitely unwanted

If not yet soon you may find out that different decoders can give you different result if it comes to counting audio file duration. It's easy to miss that milisecond type of differences. But what if this few or few hundres of miliseconds scale?

With Audiomatic you can precisly count audio files duration, as we decode everything to a raw format, count the real length and give you back undoubtly true results.

03

Many browsers - many problems...

You might think that .ogg is super cool - but is it supported by every browser? What if your users want to use .flac?

Maintaing support for multiple audio formats in a web applications is either nigthmare or demands not a small team working on it. With Audiomatic you can easily convert everything on the fly to the format acceptable for all web browser - old good pal - a MP3.

04

Messy filenames?

Most of the people doesn't care about naming their files, right? It is usually a mess but it doesn't have to be.

If the filename is a mess, there is a good chance that extracting metadata from the file can save you plenty of time.

05

Have to reach your volume knob all the time?

We know the pain. There is one simple but effective solution - ReplayGain.

Simply speaking the best way of keeping perceived loudness of a different recordings on the same level.

													class AudioFile < ApplicationRecord
													  attr_accessor :url
													  include AudiomaticRails::Model
													  #
													  # audiomatic_file
													  audiomatic_file_url :set_url
													  analyze :tags, {artist: {field: :artist_name}, album: {field: :track_title, transformation: [:capitalize]}}
													  #analyze :duration, [:duration]
													  analyze :duration, {duration: {transformation: [:to_s=]}}
													  #analyze :tags, [:artist, :title]
													  #analyze :tags, field: {title: :track_title, artist: :artist_name}

													  def capitalize result
													    result.capitalize
													  end
													  #analyze :waveform, save_to: :save_tags=


													  def set_url
													    "https://audiomatic-demo-app.herokuapp.com/audio_files/download/#{self.id}"
													  end

													  def to_s= result
													    result.to_i / 1000
													  end
													end
